PART I
IThe Problem
Why this system needs to exist and what ISRO judges care about
1.1  The world that created this problem
Over the last decade, Low Earth Orbit transformed from a frontier into a highway. Commercial mega-constellations multiplied active payloads exponentially. Alongside them: millions of pieces of debris - defunct rocket bodies, shattered solar panels, stray bolts - all orbiting at hypervelocity speeds exceeding 27,000 km/h.

This is Donald Kessler's nightmare made real. The Kessler Syndrome: a single collision at these speeds generates a shrapnel cloud that triggers cascading further collisions. Because kinetic energy scales with v², even a centimetre-sized fragment destroys a satellite and instantly creates thousands of new tracked debris pieces. A chain reaction, once started, cannot be stopped. LEO becomes permanently unusable.

◆INSIGHT: What ISRO actually cares about
ISRO's NETRA project handles collision avoidance manually. The Debris Free Space Mission (2024) targets zero new debris generation by Indian actors by 2030. The judges in the room are people who do this work today. They are not evaluating a student project - they are evaluating a potential tool.
1.2  Why the current approach is broken
BottleneckWhy it breaks at scaleScalabilityManual evaluation cannot scale. A constellation of 50 satellites may face 100+ conjunction warnings per day. Thousands of satellites face tens of thousands. There are not enough human operators.Communication blackoutsSatellites pass over oceans where no ground station has line-of-sight. If a conjunction is predicted while a satellite is in blackout, ground control is helpless. The maneuver command cannot be uplinked.Fuel sub-optimalityHuman operators plan individual burns reactively. They cannot globally optimize ΔV across a fleet of 50 satellites simultaneously. Fuel in space cannot be replenished. Wasted ΔV shortens mission lifetime.Response latencyA Conjunction Data Message is issued. Analysts evaluate it. Hours pass. The window to act may be closing. An autonomous system acts in seconds.
1.3  The problem statement - precisely
Build an Autonomous Constellation Manager (ACM) for a fleet of 50+ satellites in a debris field of 10,000+ objects. The system must:

  ▸  Ingest high-frequency orbital telemetry (ECI state vectors) via REST API on port 8000.
  ▸  Screen for conjunctions - predict collisions up to 24 hours ahead - without O(N²) computation.
  ▸  Autonomously calculate and schedule collision avoidance burns when miss distance < 100 m is predicted.
  ▸  Execute station-keeping recovery: return each satellite to its nominal orbital slot after an evasion burn.
  ▸  Track propellant strictly via Tsiolkovsky rocket equation. Schedule graveyard orbit at 5% fuel remaining.
  ▸  Serve a real-time 2D operational dashboard - the Orbital Insight Visualizer - for human oversight.
  ▸  Package everything in a single Docker container on ubuntu:22.04, binding to 0.0.0.0:8000.

PART II
IIThe Idea
What we decided to build and why
2.1  The central idea in one paragraph
AETHER is a ground-based Autonomous Constellation Manager that runs on a server and communicates with a simulation grader via REST API. It maintains an in-memory model of the entire orbital environment - 50 satellites and 10,000 debris - propagates all objects forward in time using J2-perturbed Runge-Kutta 4th order integration, screens for conjunctions using a KD-tree spatial index, and when it finds a critical conjunction, calculates the minimum-fuel evasion burn using constrained optimisation, schedules it subject to ground station line-of-sight constraints, and plans the Hohmann recovery transfer back to the satellite's nominal slot. All of this happens autonomously, in milliseconds, for every object, every time the simulation advances.

2.2  What makes this design win
DecisionWhat we choseWhy this beats the alternativesPropagatorCustom RK4 + J2 via Numba JITSGP4 is tied to TLE format - useless here. We receive raw ECI state vectors. Custom RK4 lets us control accuracy, batch shape, and JIT compile exactly the hot loop.Conjunction screeningKD-tree + golden-section TCA searchNaïve O(N²) = 500,000 pairs per tick. KD-tree reduces to O(N log N) build + O(M·k·log N) query (k≈10). 100× faster. Golden-section finds exact TCA without brute time-stepping.Collision metricAkella-Alfriend Probability of CollisionRaw miss distance ignores relative velocity. 90m miss at 0.5 km/s is safe. 90m miss at 14 km/s is catastrophic. PoC captures this. This is what real CDMs use.Maneuver planningSLSQP constrained minimisationGreedy prograde burn wastes fuel. SLSQP finds the minimum ΔV that satisfies the standoff constraint. ~30-40% fuel savings. Literature-validated approach.Recovery maneuverTwo-burn Hohmann phasingSingle retrograde burn returns altitude but not phase. Lambert solver is correct but complex to implement robustly. Hohmann phasing is the industry-standard, analytically clean, and verifiable.State managementIn-memory NumPy arraysNo database needed - state resets every Docker run. In-memory is fastest. NumPy dtype arrays give batch-compatible shapes for Numba.Frontend renderingdeck.gl on WebGLDOM manipulation fails at 10k objects. deck.gl renders 100k+ points at 60fps via GPU. ScatterplotLayer + LineLayer handles the full debris cloud.

PART III
IIIThe Design
System architecture, data flow, and layout
3.1  System overview
One process. One port. No database. No message queue. No microservices. FastAPI handles the REST layer. NumPy/Numba handles the physics. The React frontend polls the API. Everything runs inside one Docker container.

3.2  The three layers
LayerComponentsWhat it doesAPI LayerFastAPI · Pydantic v2 · UvicornReceives telemetry, validates schemas, routes to core modules, returns JSON responses. Plain def routes for CPU-bound endpoints (simulate/step). Async def for lightweight I/O routes.Core Layerphysics · state · conjunction · maneuver · planner · station_keeping · ground_station · eol · loggerThe intelligence of the system. In-memory SimState is the single source of truth. All modules read and write through it with a threading.Lock().Frontend LayerReact 18 · deck.gl · D3 · SVGFour-panel 2D dashboard. Polls /api/visualization/snapshot at 10 Hz for positions. Polls /api/status at 2 Hz for CDMs and burn schedule.
3.3  Request flow - /api/simulate/step in exact sequence
  1.  HTTP POST arrives. FastAPI routes to plain def handler in a thread pool (not async - this is CPU-bound).
  2.  Acquire sim_lock. process_due_maneuvers(sim_state, dt): apply any burns in [now, now+dt]. Apply ΔV. Run Tsiolkovsky. Mark cooldown. Release lock after.
  3.  rk4_batch(all_states, dt): propagate ALL satellites + ALL debris in one vectorized Numba call. One (M+N, 6) array goes in. One (M+N, 6) array comes out. No Python loop over objects.
  4.  sim_state.current_time_s += dt. Update ISO timestamp.
  5.  screen_conjunctions(): KD-tree coarse filter → coarse sweep → golden-section TCA → Akella-Alfriend PoC. Returns CDM list sorted by PoC descending.
  6.  run_autonomous_planner(cdms): for each CRITICAL CDM, compute optimal evasion + recovery burns, check LOS, schedule if valid. Log every decision.
  7.  eol.check(sim_state): for any satellite at ≤2.5 kg fuel, schedule graveyard maneuver, move to debris catalog.
  8.  Return: {status, new_timestamp, collisions_detected, maneuvers_executed}.

Numba decoration - exact pattern
@njit(nopython=True, parallel=True, cache=True, nogil=True)
def _batch_derivatives(states: np.ndarray) -> np.ndarray:
    N = states.shape[0]
    derivs = np.zeros_like(states)
    for i in prange(N):   # prange = parallel range, runs on multiple cores
        x,y,z = states[i,0], states[i,1], states[i,2]
        vx,vy,vz = states[i,3], states[i,4], states[i,5]
        # ... compute J2 + two-body for object i independently
    return derivs

⚠CRITICAL: Thread safety
FastAPI runs plain def routes in a thread pool. Concurrent requests can arrive simultaneously and both attempt to write sim_state. Without a lock, state is corrupted. Use: sim_lock = threading.Lock() globally. Acquire at the start of every route handler. Release at the end. No exceptions.
conjunction.py
Collision prediction - the speed-critical screening engine
Phase 1 - KD-tree coarse filter
  ▸  Build scipy.spatial.KDTree on deb_states[:, :3] - the N×3 position array of all debris.
  ▸  Query all satellite positions simultaneously: tree.query_ball_point(sat_states[:, :3], r=200.0).
  ▸  Returns a list of M lists, each containing the indices of debris within 200 km of that satellite.
  ▸  Complexity: O(N log N) build + O(M·k·log N) query, k≈10. Eliminates 99.9% of pairs.

Phase 2 - TCA search on candidates
  ▸  For each (satellite idx, debris idx) candidate pair:
  ▸    Coarse sweep: evaluate miss_distance at 300s intervals over [0, 86400s]. Find interval [t_lo, t_hi] containing minimum.
  ▸    Fine search: scipy.optimize.minimize_scalar(func, bounds=(t_lo, t_hi), method="bounded", options={"xatol":1.0}).
  ▸    Records: tca_offset_s, miss_distance_km, rel_velocity_km_s.
  ▸    Discard if TCA miss > 5.0 km. Not a genuine threat.

CDM threat levels
  ▸  CRITICAL: miss_distance_km < 1.0  - triggers autonomous burn planning.
  ▸  WARNING:  miss_distance_km 1.0-5.0  - logged, monitored, no burn.
  ▸  Both sorted by PoC descending in the returned list.

maneuver.py
Optimal burn calculation - the intelligence core
RTN frame construction
R_hat = r / |r|                    # radial: points from Earth center through satellite
N_hat = (r × v) / |r × v|           # normal: perpendicular to orbital plane
T_hat = N_hat × R_hat               # transverse: in direction of velocity
M_rtn2eci = column_stack([R, T, N]) # 3×3 rotation matrix
dv_eci = M_rtn2eci @ dv_rtn         # convert burn vector to ECI frame

Evasion burn - SLSQP optimisation
# Constrained minimisation:
# Minimize:    |dv_rtn|   (minimize total fuel cost)
# Subject to:  miss_at_tca(dv_rtn) >= standoff_km  (safety constraint)
#              |dv_rtn| <= 0.015                    (thrust limit: 15 m/s)
#              fuel_after(dv_rtn) > eol_threshold

result = scipy.optimize.minimize(
    fun=lambda dv: np.linalg.norm(dv),
    x0=np.array([0.0, 0.003, 0.0]),   # prograde bias initial guess
    method="SLSQP",
    constraints=[{"type":"ineq","fun": lambda dv: miss_at_tca(dv) - standoff},...],
    options={"ftol":1e-8, "maxiter":200}
)

  ▸  If SLSQP fails (infeasible): fallback = np.array([0.0, 0.015, 0.0]) - maximum prograde. Log as DEGRADED_AVOIDANCE.
  ▸  Fuel tiering: standoff = 0.500 km if fuel > 50%. 0.200 km if 10-50%. Skip if <10% (EOL protocol).

Recovery burn - Hohmann phasing
r_c = |sat_pos_post_evasion|  # current orbital radius (km)
r_n = |nominal_slot_state[:3]|  # nominal slot radius (km)
a_t = (r_c + r_n) / 2  # transfer orbit semi-major axis
dv1 = |sqrt(MU*(2/r_c - 1/a_t)) - sqrt(MU/r_c)|  # burn 1 (prograde)
dv2 = |sqrt(MU/r_n) - sqrt(MU*(2/r_n - 1/a_t))|  # burn 2 (prograde)
T_transfer = pi * sqrt(a_t**3 / MU)  # half-period of transfer ellipse (seconds)

planner.py
The autonomous brain - conjunctions to scheduled burns
run_autonomous_planner - exact decision sequence
  1.  For each CDM where miss < 1.0 km (CRITICAL), sorted by PoC descending:
  2.    a. Already handled? If satellite has EVASION burn for this deb_id in queue → skip.
  3.    b. EOL satellite? If sat_status == EOL → skip (eol.py handles this).
  4.    c. Too late? If tca_offset_s < 15.0 → log LATE_CONJUNCTION. Cannot act.
  5.    d. Earliest burn = max(current_time_s + 10, sat_last_burn_time + 600).
  6.    e. LOS window = predict_next_los_window(sat_state, earliest_burn). If None → log BLIND_UNRESOLVABLE.
  7.    f. LOS arrives after TCA? → log BLIND_CONJUNCTION. Cannot uplink in time.
  8.    g. Compute evasion burn via maneuver.compute_evasion_burn(). Schedule at los_window.
  9.    h. Compute recovery sequence via maneuver.compute_recovery_burn(). Schedule both burns.
  10.    i. Log CDM_ACTIONED. Set sat_status = EVADING.
  11.  For CDMs 1.0-5.0 km (WARNING): log CDM_WARNING only. No burn.

⚖LAW: Scheduling invariants
(1) No burn may have burn_time < current_sim_time + 10. (2) No two burns on the same satellite may be within 600 seconds of each other. Both conditions are checked before any ScheduledBurn is added to the queue. These are grader test cases.

PART VI
VIThe Physics and Mathematics
Every equation, every derivation, every physical constant explained
6.1  Coordinate system - Earth-Centered Inertial (ECI)
All positions and velocities in this system are expressed in the J2000 ECI frame. The origin is Earth's center of mass. The x-axis points toward the vernal equinox at the J2000 epoch. The z-axis points toward the north celestial pole. The frame is non-rotating relative to the stars - no fictitious forces (Coriolis, centrifugal) appear in the equations of motion. This is the correct frame for orbital propagation.

State vector for every object at time t:

S(t) = [x, y, z, vx, vy, vz]ᵀ    positions in km,  velocities in km/s

6.2  Equations of motion with J2 perturbation
The two-body equation (Keplerian) gives the gravitational acceleration for a point mass orbiting a spherical Earth:

d²r/dt² = −(μ/|r|³) · r

But Earth is not a sphere - it bulges at the equator. The J2 term (second zonal harmonic of Earth's gravitational potential) accounts for this oblateness. It produces nodal regression and apsidal precession - effects that accumulate and matter for conjunction assessment over 24 hours.

The J2 acceleration vector, derived from the gravitational potential expansion:

factor = (3/2) · J2 · μ · Rₑ² / |r|⁵

a_J2_x = factor · x · (5z²/|r|² − 1)
a_J2_y = factor · y · (5z²/|r|² − 1)
a_J2_z = factor · z · (5z²/|r|² − 3)

Total acceleration:
a_total = −(μ/|r|³) · r  +  a_J2

Constants:  μ = 398600.4418 km³/s²  ·  Rₑ = 6378.137 km  ·  J2 = 1.08263 × 10⁻³

6.3  Runge-Kutta 4th Order integration
We propagate the state vector forward in time by numerically integrating the equations of motion. The classic RK4 algorithm:

k1 = f(s)
k2 = f(s + 0.5·dt·k1)
k3 = f(s + 0.5·dt·k2)
k4 = f(s + dt·k3)
s_new = s + (dt/6) · (k1 + 2·k2 + 2·k3 + k4)

where f(s) = derivatives(s) = [vx, vy, vz, ax, ay, az]ᵀ.

Sub-step: 30 seconds. For a 550 km circular orbit, the period is ≈ 5760 s - about 192 steps per orbit. RK4 at this step size has global truncation error O(h⁴) ≈ negligible over 24 hours for J2-only perturbations. Euler (O(h)) would accumulate unacceptable error.

6.4  Tsiolkovsky rocket equation - propellant depletion
Every burn consumes propellant. Mass is depleted following the Tsiolkovsky equation. After each burn:

Δm = m_current · (1 − e^(−|ΔV| / (Isp · g₀)))

where:
|ΔV|  = magnitude of velocity change in km/s
Isp  = 300.0 s  (monopropellant thruster specific impulse)
g₀   = 0.00980665 km/s²  (standard gravity, converted to km/s units)
m_current  = current wet mass (dry + remaining fuel) at time of burn

As fuel depletes, m_current decreases, making subsequent burns slightly more fuel-efficient (smaller mass accelerated by the same ΔV). This is tracked per-satellite throughout the simulation.

6.5  RTN coordinate frame
Maneuver planning is done in the Radial-Transverse-Normal frame local to the satellite:

R (Radial)    : R̂ = r / |r|               - points from Earth center through satellite
T (Transverse) : T̂ = N̂ × R̂               - perpendicular to R̂, in direction of velocity
N (Normal)     : N̂ = (r × v) / |r × v|    - perpendicular to orbital plane

The rotation matrix from RTN to ECI:
M_rtn2eci = column_stack([R̂, T̂, N̂])   # 3×3 orthogonal matrix
Δv_eci    = M_rtn2eci @ Δv_rtn

Physical intuition: T-direction burns change the orbital period (phase maneuvers - cheapest). R-direction burns change eccentricity. N-direction burns change inclination (most expensive - avoid unless forced).

6.6  Probability of Collision - Akella-Alfriend model
Raw miss distance alone is insufficient as a risk metric. A 90 m miss at relative velocity 0.5 km/s is not a collision threat. A 90 m miss at 14 km/s is catastrophic. The Akella-Alfriend (2000) short-encounter model computes the probability of collision given a miss distance, accounting for position uncertainty:

r_hb   = (sat_size_m + deb_size_m) / 1000.0  # combined hard-body radius in km
sigma  = max(miss_km × 0.1,  0.010)          # 10% uncertainty floor
PoC    = (r_hb² / 2σ²) · exp(−miss_km² / 2σ²)

This is the metric used by real CDMs (Conjunction Data Messages) issued by the US Space Surveillance Network. Sorting threats by PoC rather than miss distance is the operationally correct approach. This will be recognized by ISRO judges.

6.7  Hohmann transfer - recovery maneuver
After an evasion burn, the satellite is displaced from its nominal slot. A two-burn Hohmann phasing transfer returns it. The transfer connects two approximately circular orbits at different radii:

r_c = |sat_pos_post_evasion|         # current orbital radius
r_n = |nominal_slot_pos|             # target orbital radius
a_t = (r_c + r_n) / 2               # semi-major axis of transfer ellipse

v_circ_c = sqrt(μ/r_c)              # circular velocity at r_c
v_circ_n = sqrt(μ/r_n)              # circular velocity at r_n
v_peri   = sqrt(μ·(2/r_c − 1/a_t)) # transfer orbit velocity at perigee
v_apo    = sqrt(μ·(2/r_n − 1/a_t)) # transfer orbit velocity at apogee

ΔV₁ = |v_peri − v_circ_c|           # prograde - raises apogee to r_n
ΔV₂ = |v_circ_n − v_apo|           # prograde - circularizes at r_n
T_transfer = π · sqrt(a_t³/μ)       # half-period: travel time from r_c to r_n

Both burns are prograde (T-direction in RTN) and converted to ECI via M_rtn2eci before submission. Burn 1 fires after the 600s cooldown from the evasion burn plus the next LOS window. Burn 2 fires after T_transfer + 600s cooldown + next LOS window.

6.8  Ground station line-of-sight - elevation angle
A burn command can only be uplinked when the satellite has geometric line-of-sight to a ground station. Elevation angle from station to satellite:

# Convert ground station geodetic coordinates to ECEF:
gs_ecef = (RE + alt_km) · [cos(lat)·cos(lon), cos(lat)·sin(lon), sin(lat)]

# Elevation angle (ECI ≈ ECEF approximation valid for LOS check):
range_vec = sat_pos_eci − gs_ecef
elev_deg  = degrees(arcsin(dot(range_vec, gs_ecef/|gs_ecef|) / |range_vec|))

LOS exists when elev_deg ≥ min_elevation_angle for that station. The six stations and their elevation masks are given in the PS (see Part V, ground_station.py section).
